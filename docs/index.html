Camino 2

Desarrollo Local + CLI

Editar en tu maquina y subir cambios con comandos. El flujo profesional.

Setup rapido (3 pasos)

1

Clona el repo

git clone https://github.com/irackbigsmart/hubspot-template.git
cd hubspot-template
2

Autenticate

npm install -g @hubspot/cli
hs auth

Necesitas un Personal Access Key del portal.

3

Sube y edita

hs cms upload pinit-theme pinit-theme
hs watch pinit-theme pinit-theme

Comandos esenciales

Comando Que hace
hs auth Autenticar un portal con Personal Access Key
hs accounts list Listar portales configurados
hs cms upload src dest Subir carpeta/archivo al Design Manager
hs cms upload src dest --account=ID Subir a un portal especifico (cuando tienes varios)
hs watch src dest Observar cambios y subir automaticamente
hs watch src dest --initial-upload Watch + subir todo al inicio (recomendado)
hs fetch filemanager path Descargar archivos del portal a local
hs create module nombre Crear un modulo nuevo con estructura base
hs create template nombre Crear un template nuevo

Flujo de trabajo tipico

# 1. Autenticar (solo una vez)
hs auth
# Ingresa tu Personal Access Key del portal

# 2. Subir el tema completo
hs cms upload pinit-theme pinit-theme

# 3. Activar watch para desarrollo
hs watch pinit-theme pinit-theme --initial-upload
# Ahora cada cambio local se sube automaticamente

# 4. Crear un modulo nuevo
hs create module pinit-theme/modules/nuevo-modulo

# 5. Subir un archivo especifico
hs cms upload pinit-theme/modules/hero-home.module pinit-theme/modules/hero-home.module

HubL - El lenguaje de HubSpot

HubSpot NO usa HTML puro. Usa HubL, un lenguaje de plantillas (similar a Jinja2/Twig) que se mezcla con HTML. Se ejecuta en el servidor y genera el HTML final que ve el usuario.

Variables y Expresiones

Se usan doble llave {{ }} para imprimir valores.

<!-- Imprime el valor de un campo del modulo -->
<h1>{{ module.headline }}</h1>
<p>{{ module.description }}</p>
<img src="{{ module.image.src }}" alt="{{ module.image.alt }}">

Tags (Logica)

Se usan {% %} para logica: condicionales, loops, herencia.

<!-- Condicional: solo mostrar si el campo tiene valor -->
{% if module.headline %}
  <h1>{{ module.headline }}</h1>
{% endif %}

<!-- Loop: recorrer una lista de items -->
{% for card in module.cards %}
  <div class="card">
    <h3>{{ card.title }}</h3>
    <p>{{ card.description }}</p>
  </div>
{% endfor %}

<!-- Herencia: extender un layout base -->
{% extends "./layouts/base.html" %}
{% block body %}
  <!-- contenido de la pagina aqui -->
{% endblock body %}

Estilos Dinamicos (scope_css)

Permite aplicar colores desde los campos del editor sin conflicto entre instancias del mismo modulo.

{% require_css %}
  <style>
    {% scope_css %}
      .hero-home {
        background-color: {{ module.bg_color.color }};
      }
      .hero-home__title {
        color: {{ module.title_color.color }};
      }
    {% end_scope_css %}
  </style>
{% end_require_css %}

Resumen de Sintaxis HubL

Sintaxis Uso Ejemplo
{{ }} Imprimir valores {{ module.title }}
{% %} Logica (if, for, extends, block) {% if module.title %}...{% endif %}
{# #} Comentarios {# nota interna #}
|filtro Filtros (transformar valor) {{ module.title|upper }}

Templates

Los templates definen la estructura de cada tipo de pagina. Usan DnD Areas (Drag and Drop) para que el usuario pueda reorganizar modulos desde el editor visual.

Jerarquia DnD (obligatoria)

dnd_area
→ dnd_section
→ dnd_column
→ dnd_row
→ dnd_module

Siempre respetar este orden. Un modulo NUNCA va directo dentro de dnd_area.

Ejemplo: Template Home

<!--
  templateType: page
  label: Home
  isAvailableForNewContent: true
-->

{% extends "./layouts/base.html" %}

{% block body %}
  {% dnd_area "main_content"
    label="Main Content",
    class="main-content-area"
  %}

    {# ===== Hero Section ===== #}
    {% dnd_section %}
      {% dnd_column %}
        {% dnd_row %}
          {% dnd_module path="../modules/hero-home.module" %}
          {% end_dnd_module %}
        {% end_dnd_row %}
      {% end_dnd_column %}
    {% end_dnd_section %}

    {# ===== Product Cards ===== #}
    {% dnd_section %}
      {% dnd_column %}
        {% dnd_row %}
          {% dnd_module path="../modules/product-cards.module" %}
          {% end_dnd_module %}
        {% end_dnd_row %}
      {% end_dnd_column %}
    {% end_dnd_section %}

  {% end_dnd_area %}
{% endblock body %}

Annotation (cabecera del template)

Cada template debe tener un comentario HTML al inicio con estos campos:

CampoValoresDescripcion
templateTypepage, blog_post, blog_listing, error_page, search_results_pageTipo de contenido
labelTexto libreNombre que ve el usuario
isAvailableForNewContenttrue / falseSi aparece en el selector de templates
screenshotPathRuta relativa a imagenPreview en el selector

Modulos

Los modulos son los bloques editables que el usuario arrastra en el editor. Cada modulo tiene estos archivos:

nombre.module/
fields.json ← Campos editables (lo que ve el usuario en el panel)
module.html ← HTML + HubL (la estructura visual)
module.css ← Estilos del modulo
module.js ← JavaScript (opcional)
meta.json ← Metadata: label, icon, category

Ejemplo: fields.json (hero-home)

Define los campos editables. El usuario los modifica desde el panel izquierdo del editor.

[
  {
    "name": "headline",
    "label": "Headline",
    "type": "text",
    "default": "Tecnologia para logistica"
  },
  {
    "name": "description",
    "label": "Description",
    "type": "richtext",
    "default": "<p>Soluciones tecnologicas...</p>"
  },
  {
    "name": "image",
    "label": "Hero Image",
    "type": "image",
    "default": {
      "src": "https://...",
      "alt": "TMS Pinit"
    }
  },
  {
    "name": "bg_color",
    "label": "Section Background",
    "type": "color",
    "default": { "color": "#FFFFFF" }
  }
]

Tipos de campo comunes

TipoQue generaAcceso en HubL
textInput de texto{{ module.nombre }}
richtextEditor WYSIWYG{{ module.nombre }}
imageSelector de imagen{{ module.nombre.src }}
colorColor picker{{ module.nombre.color }}
urlSelector de URL{{ module.nombre.href }}
booleanToggle on/off{{ module.nombre }}
choiceSelect / Radio{{ module.nombre }}
groupAgrupa campos{{ module.grupo.campo }}
